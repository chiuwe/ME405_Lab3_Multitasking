00027 /** \mainpage Inter-task communication tests
00028  *  This program implements a set of tests to verify that the ME405 library's 
00029  *  inter-task communication works. It's also a usable example of how the queues and
00030  *  shared data items are used. In addition, it helps to show why non thread safe data
00031  *  transfer is a bad idea; there is a global variable used to pass data between two
00032  *  tasks, and as the program runs, the global variable sometimes gets corrupted.
00033  * 
00034  *  The task diagram for this program is shown below. 
00035  *  \dot
00036  *  digraph task_diagram {
00037  *  node [ shape = box, fontname = FreeSans, fontsize = 10 ];
00038  *    task_multi1 [ label = "MultiTask 1\nPri: 1\n10-137 ms" URL = "\ref task_multi" ];
00039  *    task_multi2 [ label = "MultiTask 2\nPri: 1\n10-137 ms" URL = "\ref task_multi" ];
00040  *    task_multiN [ label = "MultiTask N\nPri: 1\n10-137 ms" URL = "\ref task_multi" ];
00041  *    task_sink   [ label = "Data Sink\nPri: 2\n0-20 ms" URL = "\ref task_sink" ];
00042  *    task_source [ label = "Data Source\nPri: 2\n1 ms" URL = "\ref task_source" ];
00043  *    task_user   [ label = "User Interface\nPri: 0\n1 ms" URL = "\ref task_user" ];
00044  *  edge [ arrowhead = normal, style = solid, fontname = FreeSans, fontsize = 10 ];
00045  *    task_source -> task_sink [ label = "queue_1\nuint32_t" ];
00046  *    task_source -> task_sink [ label = "share_1\nuint32_t", style = dashed ];
00047  *    task_source -> task_sink [ label = "glob_of_probs\nuint32_t", style = dotted ];
00048  *    task_multi1 -> task_user [ label = "print_ser_queue\nchar" ];
00049  *    task_multi2 -> task_user [ label = "print_ser_queue\nchar" ];
00050  *    task_multiN -> task_user [ label = "print_ser_queue\nchar" ];
00051  *    task_sink   -> task_user [ label = "print_ser_queue\nchar" ];
00052  *  }
00053  *  \enddot
00054  *  There are a couple of unusual features on this task diagram. The number of 
00055  *  "multi-tasks" can be varied by changing \c N_MULTI_TASKS in \c main(). These
00056  *  multi-tasks are a bunch of tasks, all created from the same class, that are used
00057  *  to put some stress on the RTOS by running lots of tasks at the same time. The
00058  *  multi-task's timing includes a delay of random duration; this timing scheme causes
00059  *  tasks to pre-empt each other with random timing. The data sink task normally runs
00060  *  very quickly, looking for changed data; if it finds data, it delays by 20 ms to 
00061  *  give other tasks some time to run. The data sink therefore takes up a lot of CPU
00062  *  time, and one can see that the user interface runs slowly because of this. 
00063  * 
00064  *  When this program is running, one can press the "v" key in a terminal window to
00065  *  cause the program to display a task status report. A typical report is shown 
00066  *  below.
00067  *  \code
00068  *  Task                            Stack
00069  *  Name            Pri.    State   Free/Total      Runs
00070  *  ----            ----    -----   ----------      ----
00071  *  Multi3          1       0       63/120          0        153 runs
00072  *  Multi2          1       0       63/120          0        161 runs
00073  *  Multi1          1       0       63/120          0        157 runs
00074  *  Multi0          1       0       63/120          0        152 runs
00075  *  UserInt         1       0       66/240          0        4217 runs
00076  *  Sink            2       0       92/160          0       1136446 runs, errors in queue: 0, shared_data: 0, global data: 265
00077  *  Source          2       0       145/220         0        10446 runs
00078  *  IDLE            0       -       50/100
00079  *  Heap: 4000/5944, OCR5A=1999
00080  *  \endcode
00081  *  This report shows tasks and their current priorities and states. Also, for the
00082  *  data sink task, it shows a report of how accurately the data from the data source
00083  *  task has been received. One should see that there have been no errors in sending
00084  *  data through the queue or shared data item, but the use of a global variable has
00085  *  caused errors due to task switching while the data was being written or read. 
00086  */
